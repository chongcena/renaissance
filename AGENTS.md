# Creative Momentum OS — Codex Instructions

## Product Definition

Creative Momentum OS is a creative operating system that turns free-flowing inspiration into organized, sustained momentum.

Do not turn this into a generic task manager, notes app, or corporate productivity dashboard.

## Core Asset Flow

Spark → Ember → Fire → Blaze

- Spark = captured creative material entering the system.
- Ember = routed potential; the asset has been interpreted, contextualized, and given possible pathways.
- Fire = active build; a chosen pathway is being developed toward an output.
- Blaze = released output; the asset has become public, useful, sold, delivered, posted, uploaded, tattooed, published, or otherwise completed outside private development.

## System Layer

Sun = repeatable engine.

A Sun is not a mandatory fifth asset stage. A Sun is a separate system layer that emerges from repeated Blazes, repeated pathways, repeated workflows, successful outputs, signature styles, product lines, service offers, content formats, IP pillars, or recurring systems.

Solar Flare = detected evidence that a repeatable Sun Engine may be forming.

## Core Concepts

- Branch = creative/business lane.
- Pathway = possible output lane for a Spark Item.
- Blaze Log = record of a released output.
- Sun Engine = confirmed repeatable system.
- Heat Score = how much attention an asset/pathway/system deserves right now.
- Heat Check = diagnostic showing what needs attention.
- Cool Down Warning = alert that something valuable is losing momentum.
- Momentum Streak = consecutive days of meaningful creative-system action.
- Burner = streak-saving token earned by meaningful output, especially releasing a Blaze.
- Burn Day = missed day protected by using a Burner.
- Freeze = intentionally parking an asset/pathway/Sun for later.
- Move = next physical action.
- Action Log = record of every meaningful action.

## Rules

- Opening the app does not count toward Momentum Streak.
- Momentum Streak is calculated only from Action Log entries with countsForStreak = true.
- Valid streak actions include capture, route, create_pathway, activate_fire, progress, complete_move, release, create_blaze, create_sun, maintain_sun, freeze, kill, archive.
- Releasing a Blaze can earn +1 Burner.
- Burn Day uses one Burner to protect a missed day.
- Freeze means parking an asset/pathway/Sun. It is not streak protection.
- Every meaningful stage/status/pathway/release/system action should create an Action Log entry.
- Stats should be generated from Action Log, Pathways, Blaze Logs, Branches, and Sun Engines.
- A single Spark Item can have multiple Pathways.
- A single Spark Item can produce multiple Blaze Logs.
- A Sun Engine can be created from one or more Spark Items, Pathways, and Blaze Logs.
- The app may detect Solar Flares but must not create Suns automatically without user confirmation.
- AI suggestions must be editable and never override the user.

## Workflow Rules for Codex

- Always work from the latest main branch.
- Do not reinitialize the app.
- Do not create duplicate scaffolds.
- Do not replace existing data/model files unless explicitly asked.
- Do not start a new PR to fix a previous conflicted PR.
- Prefer small focused changes to existing files.
- Before editing, inspect the current file structure.
- After editing, run npm run build and fix errors.
- Summarize files changed and behavior changed.
