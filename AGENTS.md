# Creative Momentum OS — Codex Instructions

## Product Definition
Creative Momentum OS is a goal-weighted creative operating system: Pillars, Goals, Schedule, Assets, Current Actions, Outputs, and Suns.

## Locked Lifecycle
Spark → Ember → Flame → Blaze

- Spark = captured idea/material/asset.
- Ember = active WIP has started.
- Flame = first released output from an asset.
- Blaze = multiple released outputs from one asset.
- Sun = repeatable system layer (not a default asset stage).

## Priority Rules
Priority is separate from lifecycle and is weighted by pillar weight, goal weight, schedule urgency, current action presence, output proximity, recent momentum/inactivity, and manual pinning.

## Terminology Rules
Use: Pillars, Goals, Schedule, Current Action, Assets, Pathways/Possible Forms, Outputs/Released Outputs, Momentum, Burners, Cooling, Frozen, Vault, Progress.
Avoid: Warm, Queued, Archive status, Fire as lifecycle label, Current Objective as primary label, Move as primary label.

## Workflow Rules for Codex
- Always work from the latest main branch.
- Do not reinitialize the app.
- Do not create duplicate scaffolds.
- Prefer focused edits using existing structure and local state.
- Do not add backend/auth/new storage unless explicitly asked.
- After editing, run npm run build and fix errors.
- Summarize behavior changes and limitations.
